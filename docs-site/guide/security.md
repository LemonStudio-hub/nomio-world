# 安全设置

本章介绍 Namio.World 的安全机制。

## 安全特性

### 强制 HTTPS

所有请求都通过 HTTPS 加密传输：
- 自动重定向 HTTP 到 HTTPS
- 使用 Cloudflare SSL 证书
- 支持 TLS 1.2+

### JWT 认证

使用 JWT（JSON Web Token）进行身份认证：
- Token 有效期 7 天
- 自动刷新机制
- 安全的存储方式

### PBKDF2 密码哈希

密码使用 PBKDF2 算法进行哈希：
- 100,000 次迭代
- SHA-256 哈希算法
- 随机盐值

### CORS 保护

严格的 CORS（跨域资源共享）策略：
- 限制允许的来源
- 防止跨站请求伪造

## 安全机制

### 输入验证

所有用户输入都经过严格验证：
- 用户名格式验证
- 密码强度验证
- URL 格式验证

### 频率限制

防止恶意请求：
- 同一发件人 5 分钟内最多 3 封邮件
- API 请求频率限制

### 文件大小限制

防止资源滥用：
- 单封邮件最大 5MB
- 邮箱存储配额 100MB

## 账号安全

### 密码安全

建议：
- 使用至少 8 个字符
- 包含大小写字母、数字和特殊字符
- 不要使用常见的密码
- 不要在多个网站使用相同的密码

### 登录安全

- 登录失败会显示错误提示
- 多次失败可能触发验证
- 建议定期修改密码

### 会话安全

- Token 存储在 localStorage
- 页面关闭后 Token 仍然有效
- 建议在公共设备上退出登录

## 数据安全

### 数据存储

- 数据存储在 Cloudflare D1 数据库
- 使用 Cloudflare 全球网络
- 自动备份

### 数据传输

- 所有数据通过 HTTPS 传输
- 使用 TLS 加密
- 防止中间人攻击

### 数据删除

- 账号删除后数据永久删除
- 邮件删除后无法恢复
- 建议定期备份重要数据

## 安全建议

### 账号安全

1. 使用强密码
2. 定期修改密码
3. 不要分享账号
4. 及时退出登录

### 域名安全

1. 验证源站所有权
2. 使用 HTTPS 源站
3. 定期检查域名状态

### 邮箱安全

1. 配置转发邮箱
2. 定期清理邮件
3. 不要公开邮箱地址

## 安全事件

### 发现漏洞

如果发现安全漏洞，请通过以下方式报告：
- GitHub Issues
- 邮件联系

### 响应时间

- 严重漏洞：24 小时内响应
- 一般漏洞：72 小时内响应

## 常见问题

### 密码存储安全吗？

是的，密码使用 PBKDF2 算法进行哈希，不可逆。

### 数据会被泄露吗？

我们采取严格的安全措施保护数据，但没有任何系统是 100% 安全的。

### 如何保护账号安全？

使用强密码、定期修改密码、不要分享账号。

### 支持两步验证吗？

当前版本暂不支持两步验证，计划在未来版本中添加。
